Rekomendacijos dėl Doclogix DVS papildomų laukų
Norint efektyvesnės AI analitikos, rekomenduojame į Doclogix MPP įvedimo formą papildomai pridėti šiuos laukus:
  • Pirkimo prioritetas (kritinis / aukštas / standartinis)
  • Strateginis projektas (priklauso projektui? kuriam?)
  • Numatoma sutarties pratęsimo galimybė (pratęstina iki ... mėn.)
  • Esamo tiekėjo žymėjimas (esamas tiekėjas / nauja rinka)
  • Bendrumas su grupės įmonėmis (požymis „centralizavimo kandidatas")
  • Sąnaudų centras / kaštų objektas (struktūrizuotas, ne tekstas)

Validacijos taisyklės

✓ Pirkimo objektas užpildytas ir prasmingas (≥10 simbolių)
✓ Pirkimo vertė be PVM > 0
✓ BVPŽ kodas pagrindinis užpildytas (8-10 simbolių formatas NNNNNNNN-N)
✓ Iniciatorius ir padalinys nurodyti

✓ Inicijavimo data ankstesnė nei pirkimo data, pirkimo data ankstesnė nei sutarties data
✓ Pirkimo būdas atitinka VPT vertės ribas (žr. „Pirkimo būdo analizė")
✓ Pirkimas per CPO suderinamas su pirkimo būdu
✓ Tarptautinis pirkimas reikalauja Valdybos pritarimo (vertė ≥ tarptautinio ribos)
✓ BVPŽ pagrindinis kodas atitinka pirkimo objekto rūšį (prekės/paslaugos/darbai)

AI centralizavimo analizėAlgoritmas analizuoja visų grupės įmonių planus ir siūlo, kuriuos pirkimus apjungti grupės lygmeniu

Kaip veikia algoritmas

1️⃣ Grupavimas pagal BVPŽ: Pirkimai grupuojami pagal pirmus 3 BVPŽ kodo skaičius (CPV3 grupė) — pvz. „302" (kompiuterinė įranga), „722" (IT paslaugos), „794" (konsultacijos).
2️⃣ Vertės agregavimas: Susumuojamos visų grupės įmonių planuojamos vertės pagal CPV3.
3️⃣ Datų derinimas: Tikrinama, ar sutarties datos sutampa per ±90 d. langą — fizinė centralizavimo galimybė.
4️⃣ Sinergijos vertinimas: Nustatomas potencialus santaupų % nuo bendros vertės (10% – maža sinergija, 30% – stipri).
